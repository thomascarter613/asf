# Review Checklist

Use this checklist when reviewing a work packet or artifact.

```text
[ ] Work matches the stated objective.
[ ] Work stays within scope.
[ ] Non-goals were respected.
[ ] Architecture boundaries were preserved.
[ ] Required files exist.
[ ] Required frontmatter exists for Markdown files.
[ ] JSON files parse where applicable.
[ ] Verification status is honest.
[ ] Failed or skipped checks are not hidden.
[ ] Safe defaults are preserved.
[ ] No forbidden tools were introduced.
[ ] No secrets were added.
[ ] Commit recommendation is appropriate.
[ ] Follow-up work is captured if needed.
```
