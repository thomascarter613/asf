# Default Work Protocol

## Purpose

This protocol defines the default execution discipline for work packets.

## Protocol

1. Read required context.
2. Confirm the work packet scope.
3. Identify affected files.
4. Make the smallest complete change.
5. Avoid scope expansion.
6. Run or record verification.
7. Update documentation if required.
8. Record events when applicable.
9. Produce handoff notes when required.
10. Identify memory candidates when appropriate.
11. Recommend an atomic Conventional Commit.
12. Do not claim completion without evidence.

## Completion Rule

Work is not complete merely because files were created or modified.

Completion requires verification or an honest record of why verification could not be completed.
