# Documentation Checklist

Use this checklist before committing documentation work.

```text
[ ] Document starts with YAML frontmatter.
[ ] Title is clear.
[ ] Purpose is stated early.
[ ] Status is accurate.
[ ] Version is present.
[ ] Updated date is current for meaningful changes.
[ ] Related ADRs or policies are referenced where useful.
[ ] Current decisions are distinguished from future work.
[ ] No placeholder prose remains.
[ ] Terminology aligns with existing documents.
[ ] Commit message is atomic and conventional.
```
