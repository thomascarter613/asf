# Directives

## Purpose

This directory stores human directives and directive records.

A directive is the raw expression of human intent.

A directive is not direct permission for unbounded implementation.

A directive should become a work packet before serious work begins.

## Rule

```text
A directive is intent.
A work packet is authorized scope.
````

## Future Shape

Future directive records may include:

1. directive ID;
2. original text;
3. author;
4. timestamp;
5. interpreted objective;
6. compilation status;
7. related work packet;
8. related run ID.

## Status

Directive records are not automated yet.

The first directive record will be created during the Initial Foundry Loop.
