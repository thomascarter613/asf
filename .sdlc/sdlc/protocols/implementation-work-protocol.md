# Implementation Work Protocol

## Purpose

This protocol governs implementation-focused work.

## Protocol

1. Start from an accepted or ready work packet.
2. Read the required context pack or required context list.
3. Confirm files to create, modify, and avoid.
4. Implement only scoped behavior.
5. Avoid silent dependency additions.
6. Avoid destructive file operations without policy approval.
7. Add or update tests when appropriate.
8. Run verification commands.
9. Record failures honestly.
10. Update documentation if behavior or structure changes.
11. Produce memory candidates for durable lessons.
12. Recommend an atomic Conventional Commit.

## Rule

Implementation is not completion.

Verification is required before completion unless explicitly waived with reason.
